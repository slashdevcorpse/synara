## Verification baseline

Repo-specific rules from `AGENTS.md`:

- Use `bun run test`, never `bun test`.
- Do not run `bun fmt`, `bun lint`, or `bun typecheck` unless the operator explicitly asks in the current conversation.
- Before marking an implementation DONE, the operator must explicitly authorize
  the final `bun fmt`, `bun lint`, and `bun typecheck` pass in that execution
  conversation, and all three must pass. Without authorization, leave validation
  pending and keep the plan IN PROGRESS.

## Findings considered and rejected

Audit run (2026-07-07) — surfaced and vetted, but NOT turned into plans this pass (recorded so they aren't re-audited from scratch):

- **SECURITY-02** (no TLS story for remote binds; session cookie lacks `Secure` at `http.ts:201`, `auth/http.ts:145`): real, MED-HIGH impact, but deferred — it pairs with 006 and needs a `--tls`/reverse-proxy design decision. Revisit right after 006.
- **SECURITY-03** (Electron `webviewTag: true` at `apps/desktop/src/main.ts:2576` with no `will-attach-webview` hardening): real, S effort, but MED confidence with no live XSS path today (markdown/file-preview renderers are properly escaped). Good cheap follow-up plan candidate.
- **SECURITY-04** (browser-use Unix socket at `apps/desktop/src/browserUsePipeServer.ts:128-181` created without `chmod 0700`): real on multi-user machines, S effort; low priority for a single-user local tool. Cheap follow-up.
- **CORRECTNESS-03** (`PiAdapter.ts:903-910` fire-and-forget `Effect.runPromise(...).catch(()=>undefined)` swallows event/log failures, unlike sibling adapters): real, S; provider-specific blind spot. Bundle into a future adapter-consistency pass.
- **CORRECTNESS-05** (`codexAppServerManager.ts:2536` writes child stdin without checking backpressure/`drain`): real, S/MED; only bites under sustained large outbound payloads. Instrumentation-level.
- **CORRECTNESS-06** (`EventNdjsonLogger.ts` `threadWriters` map never prunes per-entry; one scope/timer per thread-segment for process lifetime): real slow leak bounded by thread count; S/M. Follow-up.
- **CORRECTNESS-07** (`ProviderSessionReaper.ts:52-65` check-then-act between `activeTurnId` read and `stopSession`): real narrow race; MED-LOW confidence. Related to 007; fold into a session-lifecycle hardening pass.
- **PERF-02** (Sidebar has no list virtualization and non-memoized row closures in a 7,200-line component): real, but L effort + HIGH interaction risk (drag-and-drop, hover cards, keyboard nav). Needs its own dedicated plan; too big to bundle.
- **PERF-04** (`session-logic.ts:815-827` nine sequential `.filter()` passes over the activities array): real but constant-factor over a 500-cap array — lowest priority; can be folded into 008's follow-up activities work.
- **DEPS-01** (core framework pinned to a single `pkg.pr.new` pre-release commit `effect-smol@8881a9b` across all workspaces): real strategic risk, but the fix is "wait for a tagged release + document the accepted risk," not a code change now. Recommend documenting the pin in CONTRIBUTING.md/CLAUDE.md.
- **DEBT-01** (`ChatView.tsx` 11k lines, `Sidebar.tsx` 7.2k lines — ~100×/65× repo median): real and high-value long-term, but L/HIGH-risk; the existing hidden `.plans/03`/`.plans/04` describe this work against stale paths (`apps/renderer/`). Refresh those before executing.
